An Odd Algorithm for
Finding the Total Inside Space
of a Triangle
(O.A.F.T.I.S.T)

So, what is the purpose of this little JavaScript project?

The idea for O.A.F.T.I.S.T first began with being bored in the back of a high school calculus class. As my mind began to wander past what was being explained on the SMART Board, I began to draw. For whatever reason, instead of drafting doodles and dinky drawings, I started plotting points. More specifically, I first plotted three points randomly along a circle's circumference and attempted to find the probability you'd land within the three points, aka within the area of the resulting triangle. This idea eventually expanded into picking three random points on a Cartesian plane, drawing a triangle using those points, and then calculating the area of the triangle using only the points' coordinates.

How does O.A.F.T.I.S.T work?

  1. Use the distance formula to calculate each side length of the triangle.
  2. Create a height line between an arbitrary vertex and its opposing edge.
  3. Using Pythagorean's Theorem and some algebraic arrangement, we arrive at a formula for the length between another point and the height line.
  4. Then, using that length and a certain side length, calculate the height, and then use the old "area of a triangle" formula.

Why is this even a thing?

I honestly can't express why this discovery is significant, other than that it is represents one of the first times I have come up with some systematic solution to a weird problem by abstracting it and creating the necessary basic steps. This isn't even the first edition, either, as I first implemented O.A.F.T.I.S.T as spaghetti code in Python. I then built a quick copy in Racket (or at least one of its learning subsets) before finally building it in JavaScript so that it could live on the web.

Will there be any updates to this ... thing?

As far as the inner workings go, there likely won't be any updates. As contrived as the algorithm and title are, I'd like to think that this problem could be left to others to find a more efficient solution. As far as cosmetics go, I might very well be updating the look and visuals of this site whenever I see fit.
